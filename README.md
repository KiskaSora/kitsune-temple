# Сора — лисьи архивы

Сайт-витрина для ботов, тем и расширений SillyTavern. Чистый HTML/CSS/JS, без сборки и зависимостей — кладётся на GitHub Pages как есть и работает.

## Как выложить на GitHub Pages

Репозиторий: [KiskaSora/kitsune-temple](https://github.com/KiskaSora/kitsune-temple)
Адрес сайта: **https://kiskasora.github.io/kitsune-temple/**

Один раз включить публикацию:

**Settings → Pages → Build and deployment → Source: `Deploy from a branch` → Branch: `main`, папка `/ (root)` → Save.**

Через пару минут сайт открывается. Дальше каждый коммит в `main` обновляет его сам, ничего нажимать не нужно.

Файл `.nojekyll` уже лежит рядом — он нужен, чтобы GitHub не выкидывал папки, начинающиеся с подчёркивания. Не удалять.

**После правок обязательно поднимай цифру `?v=` в `index.html`** (две штуки — у стилей и у скрипта). От неё же зависит и подгрузка `data/*.json`: без этого браузер будет часами показывать старые данные, и будет казаться, что правка не сработала.

## Где что лежит

```
index.html            каркас страницы (менять почти не нужно)
assets/css/style.css  всё оформление; цвета — в самом верху, в блоке ПАЛИТРЫ
assets/js/app.js      сборка страниц из данных
assets/img/           картинки: превью карточек, скриншоты тем, портрет
files/                то, что скачивают: PNG-карточки, JSON тем, лорбуки
data/*.json           ВСЁ СОДЕРЖИМОЕ САЙТА
```

Правится только `data/*.json` и папки с картинками. Код трогать не нужно.

## Как залить ботов пачкой (самый быстрый способ)

Карточка SillyTavern — это PNG со встроенным описанием персонажа. Скрипт умеет его достать, так что руками ничего писать не надо:

1. Скинуть карточки `.png` в `files/bots/`.
2. Запустить:

```
python tools/import-cards.py --line pasta
```

Скрипт вытащит из каждой карточки имя, описание, первое сообщение и теги, сделает лёгкую превьюшку в `assets/img/bots/` и дополнит `data/bots.json`.

Важно: **уже заполненные поля не перезаписываются.** Если ты причесала описание на сайте, повторный запуск его не затрёт — дозаполнятся только пустые. Так что можно смело гонять скрипт после каждой новой пачки.

Ключи: `--line <id>` — в какую линейку класть новых, `--nsfw` — сразу пометить 18+, `--force-cover` — перерисовать превьюшки.

Что скрипт вытаскивает грубо и лучше поправить руками: `tagline` (берётся из заметок автора), обрезанное на 420 символов `description` и оценка `tokens` — она приблизительная.

## Как добавить бота

1. Залить превью в `assets/img/bots/` (например `mori.webp`), а саму карточку — в `files/bots/mori.png`.
2. Открыть `data/bots.json` и добавить объект в массив `items`:

```json
{
  "id": "mori",
  "name": "Мори",
  "line": "shrine",
  "tagline": "Хранитель заброшенного святилища",
  "cover": "assets/img/bots/mori.webp",
  "tags": ["мистика", "ёкай", "slow burn"],
  "nsfw": false,
  "tokens": 1480,
  "updated": "2026-07-12",
  "featured": true,
  "greeting": "Первая реплика, она же цитата на карточке.",
  "description": "Пара предложений о персонаже.",
  "files": { "card": "files/bots/mori.png", "lorebook": "files/lore/mori.json" }
}
```

Поля:

| поле | зачем |
|---|---|
| `id` | латиницей, без пробелов — из него делается ссылка `#/bots/mori` |
| `line` | `id` линейки из массива `lines` в том же файле |
| `cover` | пустая строка `""` = сайт нарисует заглушку с первой буквой имени |
| `nsfw` | `true` вешает плашку 18+ и прячет карточку под фильтром |
| `featured` | `true` = попадает на главную |
| `files` | пустая строка = кнопка станет «Файл ещё не залит» |

Новая линейка — объект в массиве `lines`: `id`, `title`, `sub`, `cover`.

## Как добавить тему или расширение

Так же, только в `data/themes.json` и `data/extensions.json`. У темы `swatches` — это цветные квадратики на карточке, обычные HEX-коды. У расширения `install` — ссылка на репозиторий, её копирует кнопка.

### Тема с расцветками

Если у темы несколько цветовых версий, они живут в массиве `variants` — тогда в карточке появляется выбор кружками, а скриншот и кнопка скачивания переключаются вместе с ним:

```json
"variants": [
  { "name": "Роза",
    "swatches": ["#1b1016", "#ecb6c6", "#f7d9e2", "#c69aa8"],
    "shots": [
      { "label": "Чат",      "path": "assets/img/themes/pearl/roza.webp" },
      { "label": "Главная",  "path": "assets/img/themes/pearl/roza-main.webp" },
      { "label": "Загрузка", "path": "assets/img/themes/pearl/roza-load.webp" }
    ],
    "file": "files/themes/pearl/pearl-roza.json" }
]
```

`shots` — массив экранов. Если он один, переключателя не будет; если несколько — над скриншотом появятся вкладки. Скриншоты показываются целиком, ничего не обрезается, так что вертикальные телефонные кадры видно от шапки до поля ввода.

Расцветки Перламутра заведены скриптом: он разобрал `custom_css` каждого файла, вытащил оттуда настоящие `--pl-bg`, `--pl-pearl`, `--pl-quote-fg` и `--pl-steel` и сжал скриншоты. Так что кружки показывают ровно те цвета, что в самой теме, а не подобранные на глаз.

Добавляешь новую расцветку — клади JSON в `files/themes/pearl/`, скрин в `assets/img/themes/pearl/` и дописывай объект в `variants`.

## Скриншоты расширений

У карточек расширений и блоков есть поле `shot` — картинка «как это выглядит», встаёт под заголовком.

Где взять: у части расширений в репозитории лежит свой `banner.svg` — он уже подставлен. Для остальных нужен обычный скриншот интерфейса из таверны.

Как добавить:

1. Снять экран с открытым расширением (можно телефоном или `Win + Shift + S`).
2. Положить в `assets/img/ext/` с именем как `id` расширения — например `sillyvn.png`.
3. В `data/extensions.json` вписать путь в `shot`: `"shot": "assets/img/ext/sillyvn.png"`.

Ждут скриншота: `sillyvn`, `sillyimages`, `avatargallery`, `floating-css`, а из блоков — `compat-lab` и `spletnik`.

Картинка показывается целиком и не обрезается, так что подойдёт любая пропорция. Держать до ~300 КБ; если скрин тяжёлый, лучше пересохранить в `.webp`.

## Гайды

Лежат в `data/guides.json`. Каждый гайд — это заголовок, обложка, теги и массив `blocks`: статья собирается из них по порядку.

Типы блоков:

| `t` | что это |
|---|---|
| `p` | обычный абзац |
| `h` | подзаголовок |
| `step` | шаг с номером в квадратике: `{"t":"step","n":1,"title":"…"}` |
| `img` | картинка: `{"t":"img","src":"…","cap":"подпись"}` |
| `code` | блок кода, переносы строк через `\n` |
| `warn` | предупреждение с красной полосой слева |

Внутри `p`, `h`, `warn` и подписей работает мини-разметка: `**жирный**`, `` `код` `` и `[текст](ссылка)`. Всё остальное экранируется, так что случайные `<` и `>` в тексте не сломают страницу.

Картинки гайдов — в `assets/img/guides/`.

**Обложка «до/после».** Если у гайда есть поле `split`, обложка рисуется из одной картинки в двух видах: левая половина размыта в мыло, правая — как есть. Подписи по углам свои:

```json
"cover": "assets/img/guides/deblur-art.webp",
"coverPos": "50% 26%",
"split": { "bad": "мыло", "good": "чётко" }
```

`coverPos` двигает кадр внутри обложки (первое число — по горизонтали, второе — по вертикали, как проценты): у портретной картинки `50% 26%` показывает лицо, `50% 50%` — середину. `"split": true` тоже работает — тогда подписи будут «мыло» и «чётко». Без поля `split` обложка обычная.

## Внешние блоки

Блоки для [ExtBlocks](https://gitgud.io/sleepydraqon/ext-blocks-custom.git) — это не расширения: они не ставятся по ссылке, а импортируются как JSON внутрь уже стоящего ExtBlocks. Поэтому у них отдельный раздел внизу страницы «Расширения», со своей инструкцией.

Живут в `data/extensions.json`, в массиве `blocks` (рядом с `items`). Файлы блоков класть в `files/blocks/`.

Поля блока: `icon` (эмодзи), `tagline`, `description`, `features` (что внутри), `perks` (мелкие фишки, выводятся строкой через точку), `steps` (нумерованная инструкция) и `note` (примечание курсивом внизу). Ссылка на сам ExtBlocks и подпись его автора — в объекте `host` там же.

`files` — массив, потому что блок может состоять из нескольких файлов. У Сплетника их два: `journalDuo-block.json` пишет номер, `journalDuoUI-block.json` его показывает. На каждый файл рисуется своя кнопка скачивания:

```json
"files": [
  { "name": "journalDuo-block.json",   "path": "files/blocks/journalDuo-block.json" },
  { "name": "journalDuoUI-block.json", "path": "files/blocks/journalDuoUI-block.json" }
]
```

## Как это делать с телефона

Всё через сайт GitHub, ничего ставить не нужно:

- **Картинки:** зайти в нужную папку → **Add file → Upload files** → выбрать из галереи → **Commit changes**.
- **Данные:** открыть `data/bots.json` → карандашик → дописать объект → **Commit changes**. GitHub сразу подсветит, если сломалась запятая или кавычка.
- Через минуту после коммита сайт обновится сам.

Совет: перед коммитом прогнать текст через любой JSON-валидатор, если правка большая. Одна лишняя запятая — и пустая страница.

## Картинки: что заливать

| куда | пропорции | размер | формат |
|---|---|---|---|
| превью бота | 3:4 (вертикальные) | ~600×800 | `.webp`, запасной `.jpg` |
| обложка линейки | 16:9 | ~1200×675 | `.webp` |
| скриншот темы | 16:10 или 9:16 для мобильных | ~1200 по длинной стороне | `.webp` |
| портрет на главной | квадрат 1:1 | ~900×900 | `.webp` |

Держать каждую картинку до ~300 КБ — сайт открывается с телефона, и гигантские PNG его убивают. Скачиваемые PNG-карточки персонажей, разумеется, заливаются как есть — они не показываются, а качаются.

## Оформление

Язык сайта — «печать»: квадратные углы, восточные уголки на карточках, двойные линейки между блоками.

- **Кнопки** — `.btn` (тонкий контур) и `.btn--acc` (главная: заливка акцентом и второй контур внутри, как оттиск печати).
- **Уголки** — вешаются классом `.corners` на любой блок; на карточках, темах, расширениях и линейках стоят сами.
- **Разделитель** — `<div class="rule">`, между соседними секциями появляется автоматически.
- **Теги** — `.tag`, цветок сакуры рисуется сам через `::before`.

Узоры (сэйгайха, кикко, облака, асаноха, веер, сакура) и уголки лежат в начале `style.css` как переменные `--p-*` и `--c-*`. Это SVG прямо в CSS: не пикселят, ничего не весят и красятся вместе с палитрой, потому что накладываются через `mask`, а цвет берётся из `background`.

Все варианты, включая невыбранные, живут на странице **kit.html** — можно открыть, посмотреть и передумать.

## Палитры

Кнопка-кружок в шапке переключает четыре палитры, собранные по цветам арта: **Ликорис** (светлая — тёплая бумага, вермильон, золото; стоит по умолчанию), **Кицунэби** (уголь и рыжий огонь), **Сакура ночью** (слива и розовый шёлк) и **Лунный храм** (холодное серебро). Выбор запоминается в браузере.

У каждой палитры свои цвета лепестков — `--petal-1/2/3`.

Все цвета — в начале `assets/css/style.css`. Чтобы поправить палитру, достаточно поменять HEX-коды в её блоке; `--glow` задаётся как три числа RGB через запятую, потому что используется в свечениях с прозрачностью.

## Локальный просмотр

Открывать двойным кликом нельзя — браузер запретит читать `data/*.json`. Нужен любой локальный сервер:

```
cd путь\к\папке\сайта
python -m http.server 8055
```

Дальше `http://localhost:8055`.
